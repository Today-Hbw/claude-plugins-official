# grill me · 提问方法论

## 你的任务

针对 PRD 需求中模糊、缺失、有歧义的部分，逐条向用户提问，直到所有关键不确定点都对齐。

## 提问策略

### 提问维度

按以下维度检查每个需求，找出需要提问的点：

| 维度 | 典型问题 |
|------|---------|
| **边界条件** | 空值怎么处理？最大/最小值？超限怎么办？ |
| **依赖关系** | 依赖哪个表/接口/其他任务？先后顺序？ |
| **存储结构** | 数据存在哪？新表还是复用？字段设计？ |
| **UI 细节** | 字段顺序？默认值？交互方式？空状态？ |
| **异常处理** | 失败怎么提示？重试？回滚？ |
| **数据迁移** | 有存量数据吗？需要迁移吗？ |
| **性能** | 数据量多大？需要分页？有并发问题？ |
| **兼容性** | 需要兼容旧版本吗？接口变更？ |

### 提问原则

1. **先读再问**：先充分理解 PRD 内容，只问 PRD 中确实没说明的
2. **逐条提问**：不要一次抛出 10 个问题，每次聚焦 1-2 个维度
3. **提供选项**：尽量给选项让用户选，而不是开放式提问
4. **给出你的判断**：先说你的理解和建议，再让用户确认
5. **标记缺口**：如果用户暂时无法回答，标记为「缺口」继续推进

### 提问方式

使用 AskUserQuestion 工具：

```
问题 1：<具体问题>
选项 A：<你的建议方案>
选项 B：<备选方案>
选项 C：<其他方案>
```

### 提问流程

```
1. 先针对第 1 个任务提问（聚焦存储/依赖）
   ↓ 用户回答
2. 针对第 1 个任务继续提问（聚焦 UI/边界）
   ↓ 用户回答
3. 切换到第 2 个任务...
   ↓
N. 所有任务的维度都覆盖后，检查是否还有遗漏
   ↓
完成 → 进入步骤 3（问答记录）
```

## 注意事项

- 如果用户说「你自己判断」，给出你的建议方案并记录
- 不要纠结于小问题，先确保大方向正确
- 如果需求文档已经足够清晰，可以快速过一遍确认，不需要强行提问
- 用户的回答可能推翻之前的理解，及时调整
