# 技术方案 · 输出格式

## 你的任务

为每个任务生成 `计划.md` 技术方案文档，写入输出目录。

## 输出格式

参考以下模板生成每个任务的 `计划.md`：

```markdown
# <任务名称>（IDxxxxxxx）· 计划

> 父需求：<所属大需求> ｜ 负责人：<人名>
> 依赖：<关键依赖说明>

## 目标
<1-3 句话说清楚这个任务要做什么>

## 任务拆解
1. **<子任务 1 名称>**（`<涉及的表/文件>`）：<简要说明>
   - <实现要点>
   - <关键代码逻辑>
2. **<子任务 2 名称>**（`<涉及的表/文件>`）：<简要说明>
   - <实现要点>
   - <关键代码逻辑>

## 边界 / 依赖
- <不做的事情>
- <依赖的外部条件>
- <需要其他人配合的事项>

## 状态：进行中
```

## 格式要求

1. **头部信息**：包含父需求、负责人、依赖
2. **目标**：简洁说清楚做什么，不超过 3 句话
3. **任务拆解**：列出所有子任务，每个子任务说明涉及的表和实现要点
4. **边界/依赖**：明确标注不做什么、依赖什么
5. **状态**：初始为「进行中」，完成后改为「已完成（详见 ../总结.md）」

## 写入规则

- 文件路径：`<output_dir>/<任务简称_ID>/计划.md`
- 任务简称从 PRD 中提取，不超过 10 个字
- 目录不存在时自动创建
- 编码：UTF-8

## 注意事项

- 方案要基于 QA 记录中的结论，不要和 QA 矛盾
- 涉及数据库变更的，要写清楚 DDL
- 涉及接口变更的，要说明前后端影响
- 如果发现方案有困难，主动和用户沟通调整
- 写完后用 AskUserQuestion 向用户确认方案
